📁 팀 프로젝트: [화학 공정 이상 탐지 시스템]

📝 프로젝트 개요

화학 공정 시뮬레이션 데이터를 기반으로 정상/비정상 상태를 탐지하는 이상 탐지 시스템을 구축한 프로젝트입니다. 센서 및 제어 변수 데이터를 활용하여 다양한 비지도 학습 기반의 모델을 비교하고, 후처리를 통해 실제 운영에서의 적용 가능성을 높이고자 하였습니다.

⏱️ 기간 및 인원

기간: 2025.04.23 ~ 2025.05.06

인원: 총 5명

🛠 기술 스택

프로그래밍 언어 및 도구: Python, Jupyter Notebook, VSCode

사용 모델: Isolation Forest, One-Class SVM, AutoEncoder, Deep SVDD

실험 관리: wandb

🔧 핵심 기능

  • ✔ t-SNE 기반 데이터 시각화로 test와 train의 분포 차이 분석
  • ✔ 센서 간 상관관계를 고려한 feature redundancy 제거 전략 수립
  • ✔ 다양한 이상 탐지 알고리즘 적용 및 재구성 오차/거리 기반 이상 판별
  • ✔ simulationRun 단위 후처리 적용으로 실전 활용도 향상

🧑‍💻 내가 기여한 부분

  • ✔ 전처리 및 feature 간 상관관계 기반 필터링 전략 설계
  • ✔ wandb를 통한 실험 자동화 및 결과 모니터링 환경 구성
  • ✔ simulationRun 단위로 이상 비율 계산 후 후처리 로직 구현

🐛 트러블슈팅

📌 문제 1: 센서 간 중복 정보로 인한 학습 성능 저하

- 문제 배경: 일부 센서 변수(xmeas_*)와 제어 변수(xmv_*) 사이에 높은 상관관계가 존재하여, 모델 학습 시 redundant feature가 오히려 noise로 작용할 우려가 있었습니다.

- 해결 방법: xmeas-xmeas는 하나만 유지, xmeas-xmv는 xmv를 우선적으로 보존, xmv-xmv는 모두 유지하는 전략으로 상관계수 기반 필터링을 수행했습니다.

📌 문제 2: Run 단위로 분리된 데이터의 평가 방식

- 문제 배경: 데이터가 시뮬레이션 단위로 구성되어 있었으나, 샘플 단위의 예측만으로는 실제 이상 여부를 파악하기 어려웠습니다.

- 해결 방법: 각 run 내에서 이상 탐지된 샘플이 5%를 초과하면 해당 run 전체를 이상으로 판단하는 후처리 로직을 구현하여, 실제 공정에서의 판단 기준에 부합하도록 설계했습니다.

  • ✔ 발표자료를 통해 각 모델의 장단점과 적용 시나리오 명확화
